SettingsController

class SettingsController(dbManager: DatabaseManager, budgetController: BudgetController, exportService: ExportService, backupController: BackupController)

Controller responsible for persisting and exposing user-configurable app settings.

Bridges the UI and DatabaseManager for all settings-related operations, including:

  • PIN lock management (enable/disable and hash-based verification).

  • Push notification preferences.

  • Display currency symbol selection.

  • PDF export via ExportService.

  • Backup and restore via BackupController.

  • Full app data reset.

Settings are stored as key-value pairs in the database using keys defined in SettingsKeys. All reactive properties are backed by Compose mutableStateOf so the UI recomposes automatically when values change.

Initialization is asynchronous: the init block launches a coroutine on Dispatchers.Default that loads all settings and then sets isLoaded to true. Gate any settings-dependent UI behind isLoaded.
