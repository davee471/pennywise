editTransaction

suspend fun editTransaction(id: Long, newAmount: Double, newCategory: Category): Result<Unit>

Edits the amount and category of an existing transaction in place.

Fetches the existing Transaction by id, applies newAmount and newCategory via Transaction.copy, then persists the updated record. The transaction type, cycle ID, and original timestamp are preserved.

Return

  • Result.Error with ErrorType.VALIDATION if newAmount ≤ 0.

  • Result.Error with ErrorType.NOT_FOUND if no transaction with id exists.

  • Otherwise, the result of the database update operation.

Parameters

id

The database primary-key ID of the transaction to edit.

newAmount

The replacement amount. Must be strictly greater than 0.

newCategory

The replacement Category to assign to the transaction.
